# AOTVI 自动化图像辅助质检系统

## 1. 项目简介

AOTVI 是一个基于 WinForms 与 .NET Framework 4.7.2 开发的 AOI（自动光学检测）图片标注系统，用于产线缺陷复判与结果上传。

系统支持扫码输入、缺陷标注、判定结果提交以及 MES 接口对接，适用于离线演示与在线产线环境。

---

## 2. 技术栈

* 平台：.NET Framework 4.7.2
* UI：WinForms
* 数据库：SQL Server
* 网络通信：HttpClient（JSON）
* 日志：log4net

---

## 3. 项目结构

```
AOTVI.UI        // 界面层（WinForms）
AOTVI.BLL       // 业务逻辑层
AOTVI.DAL       // 数据访问层
AOTVI.Models    // 实体模型
AOTVI.Common    // 公共组件（日志、配置、网络等）
```

---

## 4. 架构设计规范

* UI 层：仅负责界面展示与用户交互，不包含业务逻辑
* BLL 层：统一处理业务逻辑、异常捕获与日志记录
* DAL 层：只负责数据访问，异常向上抛出
* 外部依赖（数据库、MES）必须具备异常保护，避免 UI 卡死或崩溃
* 所有日志统一通过 LogService 输出
* 配置统一通过 App.config 管理

---

## 5. 核心功能

* 扫码输入（支持扫码枪回车触发）
* Lot 校验
* AOI 缺陷框绘制（PictureBox）
* 缺陷点击切换 NG / OK
* 数据库存取（缺陷数据加载与结果更新）
* MES 接口上传（HTTP + JSON）
* 实时日志显示（log4net + UI）
* 系统状态检测（DB / MES 连通性）

---

## 6. 运行流程

```
扫码 → Lot 校验 → 加载缺陷 → 标注判定 → 提交结果 → 上传 MES
```

---

## 7. 异常与容错设计

* 所有数据库与 MES 调用均在 BLL 层进行异常捕获
* 异常统一记录日志，并向 UI 返回可读提示
* 支持 Demo 模式：当 DB / MES 不可用时，使用模拟数据并允许流程继续执行

---

## 8. 配置说明

通过 App.config 管理关键配置：

* 数据库连接字符串（DbConn）
* MES 接口地址（MesUrl）
* 超时参数
* DemoMode（是否启用演示模式）

---

## 9. 日志机制

* 使用 log4net 记录文件日志
* 通过事件机制实时输出到 UI
* 日志级别：Info / Error

---

## 10. 注意事项

* 禁止在 UI 层直接访问数据库或调用外部接口
* 所有耗时操作必须使用异步或后台线程执行
* 提交操作需具备防重复、防误触设计
* 扫码输入需处理回车穿透与防抖问题

---
